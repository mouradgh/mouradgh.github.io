Spreadsheet

During the 4th year in Polytech Lille, we had a school projet in which we had to conceive the core of a spreadsheel application, using object oriented programming concepts in Java.
According to the specification, the outcome should satisfy these points :
- The spreadsheet contains a L x C matrix of cells. L and C may be very big, so we should only store the used cells in the memory, and not all the matrix.
- Each cell can contain either a value or a formula.
- Whe you change a cell, values that depend on it should be automatically updated.
- Circular references (When a formula refers back to its own cell, either directly or indirectly) should not be allowed.
- The user will have to possibility to save the state of the table, using serialization.

The project was carried out in 3 steps, the first was to analyze the problem and choose the suitable technical solutions, after that comes the coding and testing phase of a console application, and finally we had to program a graphic interface for the spreadsheet, using JTable.

Mobile phones comparator

At the end of my first year at Polytech Lille graduate school of engineering, I had an internship at Sofrecom (part of the Orange group). My mission consisted in developping a web application, that helps users choose the cell phone and data plan that suits them the most. The application should have a responsive design, which provides an optimal viewing experience across all devices.
The apication allows the user to find the perfect phone using two search methods:
A first search mode that 3 criteria : - Usage type (Basic -calls and texts- or advanced -Games/Internet-) - Screen size
- Phone type (Low price, best Price/Quality ratio or trendy ) It is also possible to specify the quality of the camera, battery and 4G connectivity.


The second search mode lets the user choose the brand (Samsung, Apple, BlackBerry, Nokia ...), the operating system(iOS, Android, Windows Phone ...) as well as the device type (Cell phone, tablet ou connected objects).


The main functionality of the web site is the phone comparison : The user chooses 2 of the mobile phones he's interested in, and the application performs a comparison between them, taking into account the specifications, main characteristics and reviews of each one of them.


An other important functionality is the offer recommendation engine, that allows the users to chose the data plan that suits them best, based on a mobile data use estimation.



The responsive design of the web applicaton was created using Twitter's Bootstrap.
The website is accessible without a network connection, with the help of the cache manifest, a software storage feature in HTML5.

Neutron

During my first year in Polytech Lille, one of the projects was the creation of a console application of the game "Neutron". This was an oppurtinity to practice the skills learned in both Algorithmics and C programming classes.
Neutron is a two-player abstract strategy game where each player moves two different pieces in a single turn without the use of dice.
It is normally played on a 5 x 5 board. Each player has five soldiers. There is also a neutral piece called the Neutron that is played by both players.
Each player moves the Neutron, and then one of their own pieces. All pieces move along straight line, moving as far as possible along that line (including diagonals). The only exception to this rule is the very first move, in which only a piece is moved.
LThe goal is to try and get the Neutron to your back rank, or to stalemate your opponent.
Begining of a party :

SCXML compiler

This was the final project of the Programming Languages and Translators (Principles of Compilers) course in Polytech Lille. The goal of the project was to implement a Statechart compiler. The latter should check if the SCXML automaton is valid or not, and will also provide its graphical representation.
The work consists of 3 parts :
- Defining an Attribute Grammar that checks if an XML file is in fact an automaton. - Defining a syntax-directed translation performing the following checks :
• Two states cannot have the same name
• The 'initial' attribute of the 'scxml' tag must be a reference to an 'id' attribut of a 'state' tag.
• An automaton cannot have more than 500 states and 3000 transitions.
• The attribute's string must not exceed 50 characters.
- Generating a Graphviz language formatted file, allowing the visualization of the automaton.

The first step was to analyse the problem and then chose the best data structure to store the metro stations' informations. The chosen data was an Adjacency List :

After that, you should chose a suitable shortest path's algorithme. We worked with Dijkstra's algorithme since we only have positive distances.
And last but not least, you should transform all this into an executable C program that takes as input two metro stations and returns the shortest path between them and the duration of the trip.
